## 一句话介绍

`vite-browser` 是一个面向 Vite 应用的运行时诊断 CLI，把 HMR 追踪、错误关联、传播线索和框架状态暴露成结构化终端输出。

## 适合谁

- 本地排查热更新后页面异常的开发者
- 需要把运行时状态转成 CLI 信号供 Agent 使用的人
- 想在不改项目配置的前提下检查 Vue / React / Svelte 运行时的人

## 它和常见工具的区别

- 它不是通用浏览器自动化工具
- 它不是图形化 DevTools 面板
- 它更像一个围绕 Vite runtime 行为组织起来的诊断层

## 最适合的场景

- 保存文件后 HMR 触发，页面马上坏掉
- store 更新后下游组件重渲染异常
- 当前错误和最近模块变更之间的关系不清楚
- 需要让 Agent 反复查询当前运行时证据
